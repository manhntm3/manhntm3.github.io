I rebuilt this site, again. The previous version was a single HTML file with a
terminal aesthetic that I liked for about a week. Then it started feeling like
a dorm-room poster — the kind of thing that's fun to make at 2am but doesn't
hold up the next morning.

So: new stack, new shape. Next.js for the structure, plain Markdown for the
writing, and a WebGPU black hole on the front page of the projects index for
fun. If you haven't seen it yet, [go drag it around](/projects/blackhole) —
that's a real Schwarzschild geodesic integrator running per-pixel, not a
texture.

## What I want to write about

A few directions I keep circling back to:

- **GPU programming, honestly.** Less "here's how to write a matmul" and more
  "here's the weak memory model edge case that ate three days of my life."
- **Performance engineering on real systems.** Profiling traces I actually
  looked at. The boring 80% of the work that nobody publishes.
- **Inference at the edges.** TensorRT, Triton, the gap between papers and
  what actually fits on a Jetson.
- **Side quests.** Like raymarching a black hole because I wanted to learn
  WebGPU and got curious about general relativity.

## A small example

Here's the kind of thing that's going to show up here a lot — a tight kernel
where the interesting part is one line:

```cpp
__global__ void softmax_row(const float* __restrict__ x,
                            float* __restrict__ y,
                            int n) {
  extern __shared__ float smem[];
  int tid = threadIdx.x;
  int row = blockIdx.x;

  float maxv = -INFINITY;
  for (int i = tid; i < n; i += blockDim.x)
    maxv = fmaxf(maxv, x[row * n + i]);

  // ... reduction, exp, normalization
}
```

The actual interesting part is what's missing: the reduction. I'll probably
write a post about why the obvious tree reduction is slower than the warp
shuffle version even on tiny rows, and what changes that.

## Why blog at all

Because writing forces me to actually finish thinking. Half my best ideas die
because I never had to put them in order. If even one post helps someone else
skip a debugging session, that's a fair trade.
